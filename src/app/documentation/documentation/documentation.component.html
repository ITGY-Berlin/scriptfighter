Player

The Player has all informations about the current state.
While programming you will get your own Player (me) and the enemy Player (enemy) past to the function init and the calculate next function.

Player Data Type

name type description
name string the name of the Player
hp number Player HP from 0 to 100
State PlayerState 1️⃣ The current State of the Player
position PlayerPosition 2️⃣ The current Position of the Player
finish 'WIN' | 'LOSE' | 'DRAW' Will let you know if game is over and the result of the fight
currentAction ScriptAction 3️⃣ all information about the current action in the que

PlayerState Type 1️⃣

name description
READY if the playerState is Ready, it meens the next move will directly que
STUN This will not allow the Player to do something else, the Stun need first to be finished until the next move will que and start from begining. The Player get the PlayerState STUN if he will hitted by the enemy without blocking.
BEFORE_ACTION Some actions will have a delay and while delay time the playerState is in BEFORE_ACTION state.
ACTION The action will be calculated in this playerState
AFTER_ACTION Some actions will have a recovery time and while the recovery time the playerState is in AFTER_ACTION

Position Data Type 2️⃣

name type description
isLeft Boolean will let you know, if the Player is on the left side or on the right side (isLeft = false)
positionX number Current X Position from 0 (left) to 84 (right)
positionY number Currently not used and always 0
